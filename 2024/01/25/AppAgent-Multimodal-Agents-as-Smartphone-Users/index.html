AppAgent: Multimodal Agents as Smartphone Users

AppAgent: Multimodal Agents as Smartphone Users
Chi Zhang(Tencent)
arxive
2023-12

研究问题（背景、目的及结论）

大型语言模型( LLMs )的最新进展导致了能够执行复杂任务的智能代理的产生。本文介绍了一种新颖的基于LLM的多模态代理框架，旨在操作智能手机应用程序。我们的框架使代理能够操作智能手机应用程序通过一个简化的动作空间，模仿类似人类的交互行为，如敲击和刮擦。这种新颖的方法绕过了系统后端访问的需求，从而拓宽了其在不同应用程序中的适用性。我们的代理功能的核心是其创新的学习方法。智能体通过自主探索或观察人类演示来学习导航和使用新的应用程序。这个过程就产生了代理指的是一个知识库，用于在不同的应用程序中执行复杂的任务。为了证明我们的代理的实用性，我们在10个不同的应用程序中对50多个任务进行了广泛的测试，包括社交媒体、电子邮件、地图、购物和复杂的图像编辑工具。该结果肯定了我们的代理在处理各种高级任务方面的熟练程度。

总结（结果与讨论）

在本文中，我们介绍了一种新颖的多模态代理框架，该框架利用大型语言模型的视觉能力，以类似人类的方式操作智能手机应用程序。我们的方法消除了系统后端访问的需求，并提供了安全性、适应性和灵活性的优势。我们的基于探索的学习策略允许智能体快速适应新的应用程序，具有不熟悉的用户界面，使其成为各种任务的通用工具。我们在各种应用程序上的大量实验突出了我们的代理处理各种高级任务的能力，并强调了它的适应性和学习效率。
局限性: 我们对智能手机操作采用了简化的动作空间，这意味着不支持多点触控和不规则手势等高级控制。这种局限性可能会限制智能体在一些具有挑战性的场景中的适用性。尽管如此，我们认识到这是未来研究和发展的趋势。

文章好在哪里？（创新点）

  1. 本文设计了一个Action Space，通过这个Action Space来记录下每次任务的流程，这个流程被书写成文档记录下来，以作为agent的回忆和执行任务时的参考。具体包括：

    1. Tap(element : int)
    2. Long_press(element : int)
    3. Swipe ( element : int, direction : str, dist : str)
    4. Text(text : str)/Back()/Exit()

  2. 将整个agent的框架分为两个主要部分，exploration phase 和 deployment phase。

    1. 第一个探索学习阶段又分为自主探索和demo演示学习两个部分，去制作上面提到的文档；
    2. 第二个部署阶段，给定agent一个具体任务，agent先观察UI界面并提供关于任务的当前观察的思考过程，之后agent通过调用可用的函数来执行动作，然后agent每次动作之后总结交互历史和当前执行步骤到prompt中作为agent记忆的一种，直到确定的任务已经完成执行exit()动作退出。

  3. 设计了三个参数作为评价指标：

    1. （1）successful rate：平均在10步之内完成算作成功，否者失败。
    2. （2）reward：开发了一个奖励模型来评估性能，对于应用程序中的每个任务，我们对不同的UI页面进行了评分,UI页面与目标越接近，得分越高,这意味着即使代理人未能完成任务，它仍然会根据其最终状态获得信用。
    3. （3）average steps:成功完成任务所需要的平均步骤。
